# MindLearn

MindLearn is an educational technology prototype built for the West Hacks fintech and education theme. It helps students understand why a study session may feel effective, overloaded, avoidant, or unfocused by combining self-reported check-ins, journal-based signal extraction, deterministic learning-state scoring, trend analysis, and adaptive study recommendations.

The project uses AI only for explanation and summarization after the core logic has already run. All scoring, pattern detection, trend analysis, and recommendations are rule-based and explainable.

## Problem

Students often track assignments and grades, but they rarely have a clear way to understand how stress, focus, motivation, sleep, workload, and learning behavior interact over time. A student might study for hours and still feel stuck, or avoid starting because the session feels too large. MindLearn turns those messy study signals into simple, non-clinical feedback that helps students adjust how they learn.

## Solution

MindLearn lets a student submit a daily check-in with:

- stress, focus, motivation, and energy sliders
- study hours and sleep hours
- a short journal reflection

The system then:

- extracts behavioral and cognitive signals from the journal text
- computes cognitive load, learning efficiency, and emotional friction
- detects learning patterns such as overload, avoidance loop, perfectionism, and inefficient studying
- compares current performance against recent history
- generates deterministic adaptive study strategies
- optionally uses an LLM to explain the structured learning profile in student-friendly language

## Features

- Interactive dashboard with learning-state score cards
- Check-in form with sliders and journal input
- Recharts visualization for cognitive load, learning efficiency, and emotional friction over time
- Rule-based NLP feature extraction
- Deterministic scoring and recommendation engines
- Temporal trend analysis
- PostgreSQL schema for persistent check-ins and learning states
- Demo fallback store so the app works without a database during judging

## How It Works

```text
Student check-in
  -> Feature extraction

  -> Pattern detection
  -> Temporal trend analysis
  -> Recommendation engine
  -> Optional AI explanation
  -> Dashboard and insights
```

## Core Logic

### Feature Extraction

`lib/featureExtractor.ts` uses phrase and keyword matching to produce:

- selfCriticismScore
- comparisonScore
- overloadScore
- disengagementScore

Example:

```ts
if (text.includes("should have done better")) {
  selfCriticismScore += 2;
}
```

### Scoring

`lib/scoringEngine.ts` computes:

- cognitive load
- learning efficiency
- emotional friction

These formulas are deterministic and easy to audit.

### Pattern Detection

`lib/patternEngine.ts` detects:

- Overload
- Avoidance Loop
- Perfectionism
- Inefficient Studying
- Balanced Progress

### Recommendations

`lib/recommendationEngine.ts` maps detected patterns to study strategies. Advice is not generated by AI.

## Environment Variables

Create a `.env` file if you want database persistence or AI explanations:

```bash
DATABASE_URL=postgres://postgres:postgres@localhost:5432/mindlearn
OPENAI_API_KEY=your_api_key_here
```

The app works without these variables by using built-in demo data and a local in-memory store.

## Database

Apply the schema in `db/schema.sql` to PostgreSQL.

Tables:

- `users`
- `checkins`
- `learning_state`

## AI Use

AI is intentionally limited. MindLearn does not use AI for scoring, recommendations, or pattern detection. The optional AI layer receives only structured outputs like:

```json
{
  "cognitiveLoad": 4.2,
  "learningEfficiency": 2.7,
  "detectedPattern": "Overload",
  "trend": "declining"
}
```

It then explains the result in simple educational terms without diagnosis or mental-health language.

## Hackathon Fit

MindLearn fits the education track because it gives students a practical tool for making learning more effective, organized, and reflective. It is advanced because it combines rule-based NLP, explainable scoring, temporal modeling, deterministic recommendations, and optional AI summarization in one learning-support workflow.
